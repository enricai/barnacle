<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Fastify handler → dispatch() | Validated request body crosses from HTTP layer into plugin dispatch; body has already been validated by fastify-type-provider-zod at the route layer |
| dispatch() → plugin.execute() | payload cast from unknown to TPayload; safe because Fastify/Zod validated the body against plugin.meta.bodySchema before dispatch() is called |
| dispatch() → prisma | DB writes of audit payload; payload field is user-supplied data cast to object — PII concern is noted but deferred to v2 per REQUIREMENTS.md |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Spoofing / EoP | registerRoutes() app.post() | mitigate | onRequest: [app.authenticate] preserved inside every app.post() call in registerRoutes() — never moved to enclosing scope (ASVS V4) |
| T-03-02 | Tampering | dispatch() payload cast | accept | Fastify + fastify-type-provider-zod validates body against plugin.meta.bodySchema before dispatch() is called; cast is safe post-validation |
| T-03-03 | Information Disclosure | errorHandlerPlugin + dispatch() re-throw | mitigate | VpsError re-thrown from dispatch(); errorHandlerPlugin normalizes to VPS envelope before client sees raw error |
| T-03-04 | Denial of Service | DB write failure surfaces to client (D-05) | accept | Intentional tradeoff — DB persistence is mandatory; a DB outage failing submissions is acceptable for this milestone |
| T-03-SC | Tampering | npm/pip/cargo installs | accept | No new packages installed in Phase 3; audit not applicable |
</threat_model>
